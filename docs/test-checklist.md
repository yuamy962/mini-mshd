## 一、测试前准备（必须按顺序执行）

### 1.1 配置云开发环境

| 步骤 | 操作路径 | 具体内容 |
|------|----------|----------|
| 1 | `miniprogram/app.js` | 第 8 行填入你的**云开发环境 ID** |
| 2 | 微信开发者工具 → 云开发 → 数据库 | 创建以下 3 个集合：`usage`、`quota`、`user_unlock` |
| 3 | 云开发 → 数据库 → 权限设置 | 将 3 个集合的权限设为**"所有用户可读，仅创建者可写"** |

### 1.2 配置 DeepSeek API Key

| 步骤 | 操作路径 | 具体内容 |
|------|----------|----------|
| 1 | 云开发 → 云函数 → `generateAnswer` | 点击该云函数进入详情页 |
| 2 | 版本与配置 → 高级配置 → 环境变量 | 添加变量 `DEEPSEEK_API_KEY`，值为你的 API Key |
| 3 | 部署云函数 | 右键 `generateAnswer` → **创建并部署：云端安装依赖** |

> ⚠️ **注意**：只有 `generateAnswer` 需要配置 API Key，其他云函数不需要。

### 1.3 部署所有云函数

在微信开发者工具中，依次右键以下云函数，选择**"创建并部署：云端安装依赖"**：

- [ ] `generateAnswer`
- [ ] `quota`
- [ ] `getHistory`
- [ ] `userStatus`

## 三、常见问题排查

| 问题现象 | 可能原因 | 解决方案 |
|----------|----------|----------|
| 云函数调用失败，提示 `errMsg: ...` | 云函数未部署或环境 ID 错误 | 检查 `app.js` 中的 `env`，重新右键部署云函数 |
| AI 生成超时或提示"生成失败" | `DEEPSEEK_API_KEY` 未配置或错误 | 检查云函数 `generateAnswer` 的环境变量配置 |
| 数据库读取为空或报错 | 集合未创建或权限不足 | 在云开发控制台创建集合并设置权限 |
| 次数未扣减或始终显示 3 次 | `quota` 集合权限问题 | 检查 `quota` 集合权限是否为"所有用户可读，仅创建者可写" |
| Canvas 空白或保存图片失败 | 绘制尚未完成 | 等待 1-2 秒后重试，或关闭弹窗重新打开 |
| 解锁状态丢失（重启后锁定） | 仅本地缓存，未写入云数据库 | 正常行为（测试环境），真实环境需接入 `userStatus` 云函数持久化 |
| 结果页评分模块不显示 | AI 未按要求输出 `score` 字段 | 检查 `generateAnswer` 是否重新部署（v1.1 后的新版本） |
| 推荐问题不显示 | AI 未按要求输出 `recommendQuestions` | 同上，确认云函数已重新部署 |

## 四、测试建议

1. **先用"免费体验"测试付费流程** — 等全部功能验证通过后再接入真实微信支付或广告 SDK
2. **多测几种岗位组合** — 程序员/产品经理/行政/应届生 × 不同问题，检查 AI 输出质量
3. **测试次数跨天重置** — 修改模拟器系统时间到第二天，验证次数是否恢复为 3 次
4. **测试弱网/断网情况** — 关闭 WiFi 或开启飞行模式，检查错误提示是否友好
5. **测试长文本输入** — 输入超过 200 字的背景或问题，检查前端截断和 AI 处理是否正常
6. **分享卡片内容校验** — 不同岗位和问题的卡片布局、文字换行是否正常
7. **多端测试** — 在开发者工具模拟器 + 真机预览两种环境下测试
